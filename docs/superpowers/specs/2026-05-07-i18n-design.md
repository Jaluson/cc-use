# cc-use i18n 国际化设计文档

## 背景

cc-use 目前用户可见字符串约 120 个，分布在 18 个文件中，处于中英混合状态（命令描述多为英文，交互提示多为中文）。需要引入统一的国际化机制，支持中文和英文两种语言。

## 目标

1. 支持中文（zh）和英文（en）两种语言
2. 自动检测系统 locale 作为默认语言
3. 用户可通过 `cc-use language` 命令手动切换语言
4. 手动设置优先于自动检测
5. 纯手动维护翻译，无外部依赖
6. 后续新增功能时维护成本最小化

## 非目标

- 不支持复数、日期、数字格式化等 ICU 功能
- 不引入 i18next、intl-messageformat 等第三方 i18n 库
- 不使用 AI 自动翻译
- 不预设（preset）的 `id` 和 API 相关技术术语不做翻译

## 设计

### 1. 语言包结构

采用"中文为 Key"的轻量设计。代码中直接书写中文，英文语言包作为覆盖层。

```
src/
├── i18n/
│   ├── index.ts      # t() 函数、语言检测/切换逻辑
│   ├── types.ts      # 类型定义
│   ├── zh.ts         # 中文语言包（空对象，中文为默认）
│   └── en.ts         # 英文语言包（翻译映射）
├── commands/
│   └── language.ts   # cc-use language 交互式切换命令
```

### 2. 类型约束

```typescript
// src/i18n/types.ts
export type LocaleCode = 'zh' | 'en';

export interface Translations {
  [key: string]: string;
}
```

`en.ts` 导出 `Translations` 类型对象，确保值都是字符串。不做强制的键枚举（避免维护庞大的联合类型），通过单元测试保证键的完整性。

### 3. t() 函数

```typescript
// src/i18n/index.ts
export function t(text: string, vars?: Record<string, string>): string {
  let result = text;
  if (currentLang === 'en') {
    result = en[text] ?? text;
  }
  if (vars) {
    for (const [k, v] of Object.entries(vars)) {
      result = result.replace(new RegExp(`{{${k}}}`, 'g'), v);
    }
  }
  return result;
}
```

- `text` 参数为中文原文
- `vars` 为插值变量，语法 `{{key}}`
- 当前语言为中文时直接返回原文
- 当前语言为英文时查 `en.ts`，找不到则 fallback 到中文原文

### 4. 语言检测优先级

```
1. 用户手动设置（~/.config/cc-use/lang.json 中的 locale 字段）
2. 系统环境变量 CC_USE_LANG
3. 系统 locale（os.locale()）
   - 'zh-CN', 'zh-TW', 'zh-HK', 'zh' 等 → 'zh'
   - 其他所有 locale → 'en'
4. 默认 fallback → 'en'
```

**默认语言为英文**，仅当系统 locale 明确为中文时自动切换为中文。

用户通过 `cc-use language` 命令的选择结果写入 `~/.config/cc-use/lang.json`，持久化保存。

### 5. cc-use language 命令

```typescript
// src/commands/language.ts
const choices = [
  { title: 'English', value: 'en' },
  { title: '简体中文', value: 'zh' },
];

const { locale } = await prompts({
  type: 'select',
  name: 'locale',
  message: t('选择语言 / Select language:'),
  choices,
});

saveLocalePreference(locale);
console.log(t('语言已设置为 {{lang}}', { lang: locale === 'zh' ? '简体中文' : 'English' }));
```

### 6. 预设（Preset）的多语言支持

Preset 中的 `label` 支持多语言对象：

```typescript
// src/core/types.ts 变更
label: string | Record<LocaleCode, string>;
description?: string | Record<LocaleCode, string>;
```

渲染时自动根据当前语言取值：

```typescript
function resolveLocalized(
  value: string | Record<LocaleCode, string>
): string {
  if (typeof value === 'string') return value;
  return value[currentLang] ?? value['en'] ?? Object.values(value)[0];
}
```

### 7. 代码迁移策略

分阶段替换，不一次性全改：

1. **阶段一**：搭建 `src/i18n/` 框架 + `language` 命令
2. **阶段二**：逐个文件迁移，每次一个命令文件
3. **阶段三**：迁移 `src/index.ts` 的命令描述
4. **阶段四**：迁移预设 label/description

迁移示例：

```typescript
// 迁移前
console.log(`✓ Profile "${profileLabel}" applied`)

// 迁移后
console.log(t('Profile "{{name}}" 已应用', { name: profileLabel }))
```

### 8. 测试策略

```typescript
// tests/unit/i18n.test.ts
describe('i18n', () => {
  test('t() returns Chinese when locale is zh', () => {
    setLocale('zh');
    expect(t('测试')).toBe('测试');
  });

  test('t() returns English translation when locale is en', () => {
    setLocale('en');
    expect(t('测试')).toBe('test');
  });

  test('t() falls back to Chinese if translation missing', () => {
    setLocale('en');
    expect(t('未翻译的中文')).toBe('未翻译的中文');
  });

  test('t() interpolates variables', () => {
    setLocale('en');
    expect(t('Hello {{name}}', { name: 'world' })).toBe('Hello world');
  });
});
```

## 维护指南

### 新增中文字符串

直接在代码中写中文，无需修改语言包：

```typescript
console.log(t('新的提示信息'))
```

### 给新增字符串添加英文翻译

在 `src/i18n/en.ts` 中添加：

```typescript
export default {
  // ... 已有翻译
  '新的提示信息': 'New prompt message',
};
```

### 修改已有中文

1. 修改代码中的中文
2. 全局搜索 `en.ts` 中对应条目并同步修改
